# 🏆 경쟁사(API 게이트웨이) vs 자사 솔루션(AI 보안 게이트웨이) 비교표

| 구분 | 주요 검토 항목 | **NGINX / F5** (전통적 강자) | **Kong** (API 관리의 강자) | **자사 솔루션 (Safe AI Gateway)** |
| :--- | :--- | :--- | :--- | :--- |
| **핵심 정체성** | **포지셔닝** | 네트워크 고성능 트래픽 라우팅, 방화벽(WAF) | API 통합 관리 및 서드파티 플러그인 확장 | **AI 전용 보안 가드레일 및 프롬프트 필터링** |
| **보안 탐지 방식** | **기밀 유출 방어 기술** | 정규표현식, 시그니처, 룰(Rule) 기반 매칭 | 패턴 매칭, 플러그인을 통한 일부 외부 ML 연동 | **LLM 기반 문맥 이해 탐지**<br>(정규표현식을 교묘하게 우회하는 맥락까지 파악해 차단) |
| **데이터 범위** | **멀티모달 (이미지/문서)** | 텍스트(HTTP Payload) 분석 위주 | 텍스트 쿼리 중심 분석 | **문서, 이미지, 특수 포맷(Draw.io 등)** 파일 내부 데이터를 뜯어보고 유출 검사 지원 |
| **금융 규제 대응** | **망분리 및 컴플라이언스** | 망분리 환경에서 통제 용도가 아님 (트래픽 중계만 수행) | 자체 솔루션 구축형(컴플라이언스는 고객사 몫) | 국내 금융권 특수성 반영 (망분리 예외 및 클라우드 이용 심사(협금)를 위한 **전용 프록시/PaaS형 지원**) |
| **가용성** | **대규모 트래픽 처리** | 최상급 (수만 건의 트래픽도 거뜬함) | 최상급 (API 분산 처리 능력 우수) | 망분리 프록시 구간에서 1,000명 이상 **동시 대규모 프롬프트 요청 지연(Delay) 최소화 아키텍처** 자체 제공 |
| **가치 제공** | **고객(은행)이 느끼는 혜택** | 서버다운 안 되고 네트워크 튼튼하다 | 회사 안의 수천 개 API 관리가 편해졌다 | 보안사고 걱정 없이 **임직원 전체에게 맘 편히 AI 툴(ChatGPT, 클로드)을 열어줄 수 있다** |

## 💡 전략적 킬링 타겟팅 (영업 어필 포인트)

미팅 내용에 비추어 볼 때 경쟁사 솔루션은 "네트워크 인프라(도로)"를 잘 깔아주는 거라면, 우린 "그 도로 위의 검문소(AI 가드레일)" 역할을 한다는 것을 명확히 짚어야 합니다.

1. **"그건 API 게이트웨이지, AI 게이트웨이가 아닙니다"**
   NGINX나 Kong은 뛰어난 솔루션이지만, 텍스트의 앞뒤 '문맥'을 이해하거나 '이미지' 속 개인정보를 걸러내는 (멀티모달 탐지) 지능은 없습니다. 인증/토큰 관리는 걔네한테 맡기시고, 프롬프트 보안 검열은 저희에게 맡기십시오.
2. **"망분리 규제, 저희가 같이 뚫어드립니다"**
   F5나 Kong 같은 글로벌 스탠다드 제품은 한국만의 독특한 '망분리(3호/5호 예외)' 문법을 모릅니다. 저희는 금융권 맞춤형 구조(PaaS 및 전용 프록시 아키텍처)를 통해 규제 우회와 컴플라이언스 증빙을 도와드립니다.
3. **"기존 DLP의 한계를 뛰어넘었습니다"**
   정규표현식으로 주민등록번호 잡는 건 기본입니다. 하지만 직원이 사내 시스템 도면(Draw.io) 캡처본을 통째로 올릴 때, 룰 기반 보안인 NGINX나 Kong은 무용지물입니다. 오직 당사의 솔루션만이 멀티모달 프롬프트를 인식합니다.
