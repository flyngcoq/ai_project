U+Safe AI Gateway - User Portal 통합 화면 기획 및 상세 기능 정의서 (v0.6)
본 문서는 기존의 U+Safe AI 화면기획서와 이를 바탕으로 한 상세기능정의서를 하나로 결합한 단일 통합본입니다. 화면별 핵심 정보 및 기획서에 나열된 모든 세부 UI 요소(버튼, 텍스트, 안내 배너 등) 하나하나를 단위별로 분해하여 그 의미/목적, 동작(Behavior), 표시(Display) 를 상세하게 재정의했습니다.

1. 로그인 (USR-LOGIN)
핵심 정보: 로그인 수단 제공 및 포털 진입점

1) 로고/안내 (U+Safe AI / AI Gateway — 사용자 포털, "기업 IAM 계정으로 로그인하세요" 안내)
의미: 본 포털의 서비스 정체성을 시각화하고, B2B 기업 사용자 전용 인트라넷 시스템임을 고지하여 올바른 대상자의 접근을 유도합니다.

동작: 클릭 등 별도의 인터랙션 요소는 없습니다.

표시: 화면 정중앙에 서비스 타이틀인 U+Safe AI가 크고 굵은 폰트로 표출되며, 그 하단에 작은 회색 텍스트로 보조 안내 문구가 고정 렌더링됩니다.

2) 로그인 버튼 (Alphakey, Okta, Microsoft Entra ID)
의미: 사내에서 보안 정책적으로 승인된 공식 IdP 인증 수단들과의 통합 로그인을 시작하는 핵심 트리거입니다.

동작:

각 버튼 클릭 시 해당 인증 기관(Okta/Entra ID 등)의 SSO 리다이렉트 로그인 브릿지 페이지로 이동합니다.

사용자 인증이 성공적으로 완료 및 확인되면 메인 대시보드(USR-DASH) 화면으로 자동 라우팅합니다.

표시: 각 아이덴티티 벤더사(IdP)의 공식 글로벌 로고 아이콘과 "[플랫폼명]으로 로그인" 텍스트 조합으로 이루어진 큰 прямоуго형 버튼 3개가 상하로 여백을 두고 배치됩니다. 마우스 오버(Hover) 시 버튼 배경색이 어두워지거나 쉐도우가 켜지는 시각적 피드백이 적용됩니다.

2. 접근 차단 안내 (USR-ERROR)
핵심 정보: 정책 또는 관리자에 의해 차단된 계정 접속 시도시 안내

1) 차단 안내 타이틀 및 아이콘 ("접근이 차단되었습니다")
의미: 인증/인가 처리에 실패했거나 사용자 계정이 명시적으로 중지(Suspend)된 상태임을 직관적으로 인지시킵니다.

동작: 단순 표출용 뷰이므로 별도의 사용자가 누를 수 있는 인터랙션은 없습니다.

표시: 넓은 빈 화면(Full Screen)의 정중앙 중앙 정렬로, 붉은색 방패 또는 엑스(X) 형태의 큰 아이콘과 함께 헤딩(H1급) 텍스트로 "접근이 차단되었습니다" 문구가 노출됩니다.

2) 상세 사유 및 문의처 ("계정이 관리자에 의해 차단되었습니다. 문의: it-support@uplus.co.kr")
의미: 차단된 사용자가 오해하거나 당황하지 않도록 명확한 차단 사유를 노출하고, 소명이나 문제 해결을 위해 연락할 수 있는 기업 내 IT 관리 데스크 이메일을 가이드합니다.

동작: 이메일 주소 영역 텍스트를 마우스 혹은 터치로 클릭하면, 사용자 OS(윈도우/맥)의 기본 설정된 메일 클라이언트 프로그램(예: Outlook)이 즉시 발송 창(mailto:) 상태로 트리거 및 오픈됩니다.

표시: 위의 헤딩 타이틀 바로 아래 조금 더 얇고 작은 일반 폰트를 사용하여 부연 설명으로 배치되며, it-support@uplus.co.kr 이메일 문자열 자체만 파란색(Blue) 하이퍼링크 텍스트 형식으로 시각적 돋보임 처리를 합니다.

3) 액션 버튼 ("로그아웃")
의미: 현재 유효성이 없는 세션을 명시적으로 파기하고 정상적인 초기 로그인 화면으로 세션을 클리어하여 되돌리기 위한 탈출 동작입니다.

동작: 클릭 시 로컬 브라우저에 저장된 토큰/사용자 쿠키 캐시 데이터를 모두 파기 후 로그인(USR-LOGIN) 화면으로 즉시 리다이렉트 처리합니다.

표시: 이메일 문의처 텍스트 하단에 일정한 여백을 두고, 회색 톤이 섞인 서브(Secondary) 형태의 눈에 띄는 버튼 UI로 렌더링됩니다. 화이트 테마에 윤곽선 둥근 모서리로 제작됩니다.

3. 세션 만료 안내 (USR-SESSION-EXPIRE)
핵심 정보: 세션 타임아웃 발생에 따른 재로그인 유도

1) 만료 안내 타이틀 및 아이콘 ("세션이 만료되었습니다")
의미: 정상적으로 접속해 있었으나, 장기간(통상 30분~4시간 등 조직 정책 기반) 마우스 움직임이나 API 콜 발생 등 활동이 없어 강제 타임아웃 처리된 상태를 고지합니다.

동작: 클릭 시 피드백 없습니다.

표시: 화면 정중앙에 위치하며, 붉은 에러 아이콘보다 덜 심각함을 알리기 위해 노란색 경고(경광등 기호나 정지된 시계 기호 등) 아이콘과 함께 큼지막한 텍스트로 노출됩니다.

2) 안내 문구 ("보안을 위해 일정 시간 후 자동 로그아웃되었습니다. 다시 로그인해 주세요.")
의미: 사내 망 보안 정책상 세션을 닫았음을 구체적으로 설명하여 시스템 에러나 버그로의 오인(CS 접수)을 방지합니다.

동작: 없음

표시: 헤딩 아래 중앙 정렬 안내 텍스트로 보이며, 가독성을 고려한 줄바꿈을 지원합니다.

3) 액션 버튼 ("다시 로그인")
의미: 사용자가 업무를 이어서 하기 위해 재인증을 즉시 수행토록 유도합니다.

동작: 클릭 시 로그인(USR-LOGIN) 화면으로 되돌아갑니다. (가급적, 만료 전 작업 중이던 라우팅 주소 파라미터 ?redirect=/usr-apply-list 등을 보존하여 인증 성공시 바로 점프하게끔 프론트 처리 권장)

표시: 강렬한 파란색 또는 브랜드 고유의 Primary 컬러 단색이 칠해진 시인성 높은 버튼으로 중앙 맨 아래 배치됩니다.

4. 대시보드 (USR-DASH)
핵심 정보: 사용자의 주요 현황 및 진척을 한눈에 확인할 수 있는 Home 화면

1) 환영 메시지 ("안녕하세요, [사용자명]님 👋")
의미: 정상적인 인가 및 개인화된 포털 뷰임을 신속하게 인식시키고 환영의 뜻을 표명합니다.

동작: 요소 자체 인터랙션은 없습니다.

표시: 메인 컨텐츠 영역의 좌측 최상단에 큰 타이틀 공간을 차지하며 볼드체로 텍스트와 손흔드는 인사 이모지가 들어갑니다. DB에 연동된 이름 정보(예: "김철수")가 동적으로 매핑됩니다.

2) 사용 현황 요약 카드 (Stat Cards)
의미: 복잡한 표를 보기 전에 사용자의 필수 권한/승인 상태/비용 지표 4가지를 최우선 압축하여 노출하는 요약 보드입니다.

동작: 각 개별 타일 박스 위를 클릭하면, 타일 성격에 맞는 해당 상세 페이지(USR-APPLY-LIST, USR-USAGE 등)로 퀵 라우팅 이동하는 기능이 수행됩니다.

표시: 사용자명 아래로 4열 그리드(디스플레이가 좁아지면 가변형 2행 2열 등)로 동일한 크기의 사각형 카드들이 수평 배열됩니다.

[승인된 AI Tool]: 사용 중인 개수를 중앙에 거대 폰트로 출력. "사용 중" 보조 라벨 노출.

[대기 중인 신청]: 결재를 기다리는 숫자를 출력. "관리자 검토 중" 보조 라벨.

[이번 달 예산 사용량]: 이번 달에 소비된 누적 금액($ 표기) 데이터를 큰 중앙 수치로 표시. "한도: $NNN" 문자열 보조 추가.

[처리 완료 신청]: 총 누적 반려/승인을 포함해 프로세스 종결된 수를 출력 및 "누적 승인" 표현.

3) 이번 달 툴별 예산 사용량 (Multi Progress Bars)
의미: 사용자가 여러 AI Tool을 동시에 사용할 경우, 전체 예산뿐만 아니라 각 Tool(Provider)별로 예산을 얼마나 소모하고 있는지 직관적으로 개별 파악하게 하여 자가 통제 및 남용 방지를 돕습니다.

동작: 특정 툴의 예산 사용률이 80%를 넘어가는 위험 임계치에 도달하면 해당 막대의 컬러가 붉은 톤으로 동적 전환됩니다.

표시: 스탯 카드 하단에 사용 중인 각 AI Tool 개수만큼 여러 개의 얇은 게이지형 바(Bar) 영역이 상하 목록(List) 형태로 나열됩니다.

각 바 조측 상단(또는 내부)에는 해당 AI Tool 명(또는 Provider) 표기.

바의 채우기 컬러는 기본: 초록색, 경고(80%이상): 주황색, 가용 한도초과(100%): 붉은색으로 동적 렌더링.

각 바 우측 상단에 파란색 Info 라운드형 사용 % 뱃지 표시.

각 바 하단에 양갈래 텍스트로 좌측은 "현재 사용 예산 / 한도 예산 ($ 표기)", 우측 영역은 "잔여 $NNN" 형태로 노출.

[예외 처리] 예산 한도가 설정되지 않은 툴(무제한)일 경우:

비율(%) 뱃지 대신 '∞ 무제한' 텍스트를 담은 회색 뱃지를 노출합니다.

프로그레스 바(Bar)는 게이지 폭(Width)을 100% 고정하되 시각적인 위험도를 주지 않기 위해 무채색(연한 회색 등) 배경으로 렌더링합니다.

하단 텍스트는 좌측 "현재 사용 예산 / 한도 없음", 우측 영역은 **"제한 없음"**으로 처리합니다.

4) 나의 AI Tool 목록 (간단 보기)
의미: 신청이 완료되어 현 시점 즉시 엔드포인트 연동이 가능한 AI 플랫폼(GPT, Claude 등)의 상태 인지를 브리핑합니다.

동작: 내역 우상단의 [+ 신청] 텍스트 또는 '+' 아이콘 버튼 영역 클릭 시 즉방향으로 신규 신청 폼(USR-TOOL-APPLY) 으로 페이지 라우팅 됩니다.

표시: 카드 컨테이너 내부에 작은 표(테이블) 형태로 제공됩니다.

Tool명 (굵은 글씨)

Provider (제조사 로고나 뱃지)

상태 (초록색 Active 고정 뱃지 라벨)

사용 기간 안내 (~2026.12.31 형태로 고지)

5) 최근 신청 내역 (간단 보기)
의미: 본인이 최근 품의 올린 결재건들이 처리되었는지 빠르게 추적하는 용도입니다. (가장 많이 확인하는 화면 중 하나)

동작: 특정 행(로우) 항목 하나 클릭 시 해당 건에 대한 신청 상세(USR-APPLY-DETAIL) 팝업 모달 또는 화면 넘어가기가 진행. 블록 우상단의 [전체 보기] 텍스트 버튼 클릭 시 전체 신청 내역 목록 (USR-APPLY-LIST) 로 이동합니다.

표시: 나의 AI Tool 우측 편 그리드로 배치되며 심플한 3개 열의 표가 렌더링됩니다.

신청일 (회색 텍스트), AI Tool명

승인 상태: Pending(대기, 회색배경), Approved(승인, 초록/파랑배경), Rejected(반려, 빨간배경) 뱃지가 박혀 직관적인 상태 파악 가능 안내. 리스트는 최대 3~5줄 선에서 자릅니다(Cut-off limit 적용).

5. AI Tool 목록 (USR-TOOL-LIST)
핵심 정보: 플랫폼 내 신청 가능한 전체 AI Tool 목록 조회, 검색 및 신청

1) 탐색 영역 ("Tool명 검색..." 필터 필드 제공)
의미: 회사내 백여개 이상의 AI 서비스나 에이전트/툴이 연동될 경우를 대비해 본인이 찾고싶은 서비스(Cursor 등)를 재빠르게 추려내기 위한 공간입니다.

동작: 사용자가 키보드 타이핑하여 글자 값을 입력할 때마다(Input 이벤트 감지), 실시간(Debounce 적용)으로 즉각 필더링 동작을 거쳐 하단 카드 갯수가 줄어듭니다. Enter 없이 작동합니다.

표시: 메인 헤더 아래 넓게 가로를 채우는 텍스트 입력창 Input Box이며 좌측에 돋보기 형태의 아이콘이 회색 톤으로 들어있어 검색 창임을 유추하게 합니다. 내부에 Tool명 검색... 안내 박혀있습니다.

2) Tool 카드 리스트
의미: 각 AI 도구가 어떤 속성인지 구체적인 기능과 서비스 내용을 그리드 타일 형태로 전시합니다.

동작: 본문 전체 카드 박스 영역(상태 버튼 제외)을 클릭할 경우 중앙으로 간단한 툴 상세/기능 소개 툴팁 팝업이 부드러운 애니메이션과 함께 오픈하게 됩니다 (팝업 기능이 구현될 시).

표시: 화면 전체에 3개~4개 다단 형태로 카드 모양 Grid 배열됩니다.

AI Tool 명 (두꺼운 볼드 폰트)

Provider 명시 라벨 (OpenAI 등)

Tool 설명 문자열: "Anthropic의 AI 코딩 에이전트. 터미널 기반에서 백그라운드로 작동 등" 특징을 2~3줄 가량의 회색 본문으로 나열.

상태 뱃지: 사용자의 권한을 맵핑해 "사용 중"일 경우 초록색 Active 뱃지, "미신청"일 경우 속이 빈 Inactive 뱃지, 아이콘 심볼도 포함시켜 시각 차단을 합니다.

3) 액션 버튼 ("신청하기" 또는 "이미 사용 중")
의미: 사용자가 해당 툴에 대해 갖는 권한 상태에 따른 동적 액션(Call To Action)을 유도합니다.

6. AI Tool 사용 신청 (USR-TOOL-APPLY)
핵심 정보: 신규 AI Tool에 대한 사용 권한 신청 폼

동작: 고정된 레이아웃 형태입니다. 좌측 X(닫기) 클릭 시 닫히기도 합니다.

표시: 타이틀 영역 바로 하단에 긴 파란색 바(Blue Banner) 형태로 "i(info)" 아이콘과 해당 문자열들이 띠 모양으로 둘러집니다.

2) 입력 폼 필드
AI Tool 선택 (Select Box 옵션)

의미: 대상 소프트웨어 지정

동작: 드롭다운 오픈 ➡ 옵션 클릭 선택 (일반적인 select 엘리먼트 폼 작동). 이전 목록 페이지에서 '신청'버튼을 눌러 넘어온 상태라면 기본값(Value)이 목적지 Tool에 자동 세팅되어야 함.

사용 기간 선택 (1/3/6/12개월 Radio Button 옵션)

의미: 퇴사자/이동자 등에 대한 무분별한 종신 권한 및 비용 지출 방지 위한 유효기간 선택

동작: 오직 하나만 단일 토글 클릭 가능 (기본값으로 안전한 '3개월' 이 초기 선택됨)

표시: 가로형으로 라디오 원형 버튼(Radio)이나 토글 버튼(Toggle Buttons) 스타일의 4가지 초이스 버튼 노출.

부서 정보 (읽기 전용 자동 매핑)

의미: 인사 DB 연동으로 현재 결재 주체 본인의 실 부서가 자동 태깅됨을 가이드하여 추후 비용 청구 단위(과금 부서) 인지 향상.

동작: 사용자가 값 타이핑/변경/클릭 일절 불가능 한 비활성화 상태 (Read-only/Disabled).

표시: 회색 배경색에 글자는 약간 투명한 검은색으로 입력 필드 쉐이프로 노출됨. (예: "플랫폼 개발부서")

3) 경고 배너 ("개인정보, 기밀 정보 입력 금지 (AI-DLP 탐지)" 정책 경고)

취소: 클릭 시 폼 값이 초기화되고 바로 이전 진입 화면 주소(History Back)로 연동 복귀.

7. 나의 신청 내역 (USR-APPLY-LIST)

2) 상태별 필터 탭 (전체 / 대기 중(알림 뱃지) / 승인 / 반려 카테고리 필터링)

표시: 화면 중상단 위치 수평 가로 막대기형 탭(Tab) UI.

각각의 글자가 보이며 대기 중 파트 우측 어깨 언저리에 건수를 가리키는 빨간색 알림 동그라미 뱃지(Badge) 노출 (예: 숫자 2).

동작: 각 컬럼 데이터의 헤더(신청일 등)를 클릭 시 ASC(오름차순)/DESC(내림차순) 정렬 처리. Row 맨 우측 영역 액션 영역에서 [상세] 버튼 클릭하면 구체화된 내용 뷰USR-APPLY-DETAIL 로 점프.

신청일, AI Tool, Provider 컬럼

4) 하단 영역 (총 건수 및 페이지네이션)

8. 신청 상세 (USR-APPLY-DETAIL)
핵심 정보: 단일 신청건에 대한 자세한 정보, 진행 상황 및 관리자 피드백 확인

표시: 모달 또는 화면 상단 파트에 2컬럼 다단 양식 그리드 표로 속성값(Key-Value) 형태로 배치됨. 특히 상태값은 회색/파랑 등 직관적 라벨박스로 튀게 처리.

검토 중(대기) 일 경우 안내 배너

동작/표시: 노란색 또는 회색 인포 배너 UI 렌더링. "관리자 검토 중입니다. 1~2일 영업일 정도 대기가 필요합니다" 모래시계 등 애니메이션 아이콘.

반려되었을 경우 ('사유 텍스트 노출', '처리일', '처리 관리자', '재신청하기' 액션 활성화)

표시: 두껍고 경보 색이 있는 붉은 섹션 추가. 관리자가 타건한 텍스트 값이 노출. 우하단에 프라이머리 재신청하기 버튼 추가됨.

9. 나의 토큰 / AI Gateway 사용 (USR-TOKEN-VIEW)

의미: 사용자가 용도별로 발급한 여러 User Token들의 활성화 상태 및 생명주기를 한눈에 요약하고 보안 위험을 관리하는 제어 테이블.

동작 및 통제(보안 정책):

"마스킹 해제(점 보기) 기능을 제공하지 않음": 모든 테이블의 토큰 문자열은 ugw_tok_...8dAA처럼 강력하게 부분 마스킹 처리된 읽기 전용 텍스트 상태로 남게 됩니다. (글로벌 보안 표준 적용)

토큰을 잊었거나 탈취가 의심되는 경우, 기존 토큰을 테이블에서 직접 '삭제(폐기)'하고 완전히 새 토큰을 발급받도록 강제하여 보안성을 높입니다.

표시: 포털의 표준 테이블 형태로 렌더링하며 다음 컬럼 항목을 가집니다.

토큰 이름: 사용자가 지정한 식별 텍스트 (예: 'CI/CD 자동화 봇용')

상태: 유효(Active) 등 상태 뱃지 표시

Token Key: 고정 마스킹 처리 텍스트

발급일 / 마지막 사용일: 생명주기 및 도용 확인용 타임스탬프

관리 액션: ✏️(이름 수정) 아이콘 및 🗑️(삭제/폐기) 아이콘 클릭 시 관련 동작 수행.

구성: 빨간색 에러 배너로 "정말 이 토큰을 폐기하시겠습니까? 사용하는 애플리케이션의 인증이 영구 중지됩니다."라는 경고를 전시. 대상 토큰의 이름과 마스킹 키를 한 번 더 표기. [취소], 빨간색 [폐기] 액션 버튼 체계.

4) 빠른 시작 가이드 (cURL 예시 코드 기반)
의미: 이 문서를 확인 즉시 개발자가 복잡한 래퍼런스 안가고 윈도우 터미널(CMD) 켜서 호환성 API 연동 테스트가 작동하는지 퀵하게 구동할 개발 경험(DX) 지원.

동작: [전체 가이드 보기] 버튼 누르면 보다 전문적인 API 연동 가이드 (USR-GW-GUIDE) 화면으로 풀 이동됨.

표시: 터미널과 동일한 문법 컬러링 요소(Syntax Highlighter)가 들어간 작은 박스 블록.
