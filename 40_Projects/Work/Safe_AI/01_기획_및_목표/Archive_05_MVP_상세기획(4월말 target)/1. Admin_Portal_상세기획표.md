# 🛡️ U+Safe AI Gateway - Admin Portal 상세 기능 정의서 (v0.6)

본 문서는 `U+Safe_AI_화면기획_어드민.md`에 정의된 주요 관리자 화면을 토대로, 세부 UI 요소(버튼, 텍스트, 안내 배너 등) 하나하나를 속성 단위(의미, 동작, 표시)로 매우 구체적으로 맵핑하고 텍스트로 정의한 종합 산출물입니다.

---

## 0. 공통 레이아웃 및 UX 편의 기능 (App Layout & Common UX)
> **핵심 정보:** 관리자 포털 전반에 걸쳐 제공되는 탐색 편의성 및 공간 활용성 제공 기능

### 1) 네비게이션과 현재 위치 파악 (LNB & Breadcrumb)
* **의미:** 복잡한 관리자 메뉴 트리에서 현재 위치를 잃지 않도록 돕습니다.
* **동작 및 표시:**
  * **활성화 상태 표시(Active State):** 좌측 사이드바(LNB) 메뉴 클릭 시 배경색상 반전과 함께 폰트 두께가 변화하여 활성 메뉴를 직관적으로 강조합니다.
  * **경로 표시(Breadcrumb):** 최상단 우측 여백 혹은 타이틀 바로 아래에 `Home > 비용 관리 > 비용 한도 설정` 형태로 현재 진입 깊이를 텍스트로 보조 표시합니다.

### 2) 화면 공간 활용성 (LNB Accordion Toggle)
* **의미:** 가로로 긴 데이터 테이블(표)이 많은 관리자 화면을 효과적으로 시청하기 위해 좌측 메뉴 뷰포트를 축소합니다.
* **동작 및 표시:** 로고 옆 `[◀▶]` 사이드바 접기 토글 버튼 클릭 시, 텍스트 메뉴가 전부 Display None 처리되며 아이콘만 세로 방향으로 나타나는 미니 LNB 형태로 면적이 축소됩니다.

## 1. 로그인 (ADM-LOGIN)
> **핵심 정보:** 관리자용 로그인 수단 제공 및 포털 진입점

### 1) 로고/안내
* **의미:** 본 화면이 시스템 설정과 통제가 이루어지는 고위험 권한(최고 관리자) 구역임을 인지시킵니다. (안내문: "관리자 IAM 계정으로 로그인하세요")
* **동작:** 클릭 등의 인터랙션 제어 요소는 없습니다.
* **표시:** 일반 사용자 포털과 형태는 동일하나, 배경 테마가 훨씬 진하거나 다크 모드 형태로 배색되어 관리자 화면임을 직관적으로 시각 차별화(Color Scheme)합니다.

### 2) 로그인 버튼
* **의미:** 사내 승인된 IdP 연동(SSO)을 통한 고신뢰도 관리자 인증 체계 브릿지입니다. (`Alphakey`, `Okta`, `Microsoft Entra ID`)
* **동작:**
  * 각 버튼 클릭 시 해당 인증 기관 로그인 페이지로 리다이렉트 됩니다.
  * 일반 유저 권한 계정으로 로그인 성공 시 차단(ADM-ERROR) 화면으로 강제 이동 처리되며, "최상위 권한자(Super User/Admin)" 인증 확인 시 메인 관리자 대시보드(ADM-DASH)로 접근을 허가합니다.
* **표시:** 공식 벤더 로고 및 버튼 UI 표출.

## 2. 접근 차단 안내 (ADM-ERROR)
> **핵심 정보:** 권한 없음 판단, 정책 또는 상위 관리자에 의해 차단된 시스템 관리자 접속 시도시 안내

### 1) 차단 안내 타이틀 및 아이콘
* **의미:** 비인가 접근, 혹은 권한 박탈에 대한 강력한 알림 시스템.
* **표시:** 화면 중앙 H1급의 검고 거대한 굵은 텍스트("접근이 차단되었습니다") 및 강렬한 붉은색의 금지(🚫) 아이콘이 조합됩니다.

### 2) 상세 사유 및 문의처
* **의미:** 관리자 그룹 내에서 왜 퇴출되거나 반려/정지 당했는지의 맥락과 기술팀 핫라인(Super Admin)을 노출합니다.
* **동작:** 이메일 파란색 텍스트 클릭 시 시스템 메일 모듈(`mailto:`) 트리거.
* **표시:** 이메일 주소만 돋보이는 보조 텍스트 렌더링. ("계정이 관리자에 의해 차단되었습니다. 문의: admin@lguplussafe.com")

### 3) 액션 버튼
* **의미:** 브라우저에 잔류하고 있는 찌꺼기 세션 쿠키를 폐기하고 안전 라인으로 퇴장 조치합니다.
* **동작:** 1개의 [로그아웃] 버튼. 현재 발급된 관리자 Token 데이터 소거 후 다시 로그인 창(ADM-LOGIN) 리다이렉트.
* **표시:** 중앙 하단 프라이머리 버튼(회색/파랑 등).

---

## 3. 세션 만료 안내 (ADM-SESSION-EXPIRE)
> **핵심 정보:** 타임아웃 발생에 따른 재로그인 유도 (보안 유지)

### 1) 만료 안내 타이틀 및 아이콘
* **의미:** 콘솔 뷰어에 장시간 유휴(Idle) 상태가 유지되어 보안 정책에 의해 자동으로 토큰이 절단되었음을 설명합니다. (장애가 아님)
* **표시:** 다소 부드러운 노란색 시계(⏰) 등 주의 아이콘과 함께 "세션이 만료되었습니다" 헤딩 문구 노출.

### 2) 안내 문구 및 액션 버튼
* **의미:** 맥락 설명과 재탈환 유도. ("보안을 위해 자동 로그아웃...")
* **동작:** `[다시 로그인]` 버튼 클릭 시 ADM-LOGIN 화면으로 회귀합니다.
* **표시:** 텍스트 설명 아래 1개의 버튼 배치.

## 4. 관리자 대시보드 (ADM-DASH)
> **핵심 정보:** AI Gateway의 전체 사용자, API 인프라 통계 현황 통합 뷰어

### 1) 데이터 기준일
* **의미:** 시스템 통계의 Cache가 언제 마지막으로 DB에서 갱신되었는지 현행성을 고지합니다.
* **동작:** "새로고침(Refresh)" 버튼의 경우 수동 갱신 통신을 발생시킴.
* **표시:** 헤더 우측 작고 희미한 타임스탬프 문자로 렌더링. ("최종 업데이트: YYYY.MM.DD HH:MM")

### 2) 요약 카드 (Stat Grid 6종)
* **의미:** 어드민이 출근해 필수 점검해야 할 핵심 인덱스 6개 압축 현황 지표입니다.
* **동작:** 클릭 시 각각의 디테일 화면(ADM-USER-LIST, ADM-APPLY-LIST 등)으로 즉시 단축(Shortcut) 라우팅 연동.
* **표시:** 상단 3개, 하단 3개 등 6분할 사각 카드 타일 뷰포트 배치. 수치는 모두 크고 굵은 타이포그래피.
  * **전체 사용자 현황:** 총 유저수 중 괄호로 `(활성 N / 차단 M)` 표기.
  * **대기 중인 신청:** 붉은 뱃지로 펜딩 숫자(예: 15건) 표기.
  * **Provider API Key 현황:** 전체 수량 중 `(서빙 중 O, 비활성 P)` 표시.
  * **이번 달 비용 현황 ($):** 예산 배정액 대비 현재 연소액(`$450 / 80% 소비됨`).
  * **DLP 탐지 건수:** 전월 대조기호 추가(`22건 ▲15%`).
  * **누적 호출 트래픽/토큰:** 45M Tokens 등 축약된 Million/K 단위 표기.

### 3) 최근 AI Tool 신청 현황 요약 보드
* **의미:** 관리자의 최우선 업무(수동 결재)인 신청 목록 중 최근 TOP 5건을 대시보드에서 선제 브리핑.
* **동작:** 각 아이템의 `[검토]` 클릭 시 바로 상태 변경 뷰(ADM-APPLY-DETAIL) 진입 및 전체 보기 라우팅.
* **표시:** 가로축 배열 박스형 테이블 내열 리스트. (이름/Tool/날짜/Badge 표출)

### 4) 이벤트 / 로그 뷰어 (최근 활동 및 DLP 보안 이벤트 요약)
* **의미:** 시스템 상태와 이상 징후, 관리 기록 모니터링 레이더.
* **동작:** 차트 내 라벨 버튼(개인정보 탐지 N건 등) 클릭 시 DLP 및 보안 로그 뷰어로 필터 연동.
* **표시:** 하단 가로지르는 2열 배치. 좌측은 세로형 타임라인 UI, 우측은 탐지 위협군 차트 그리드.

## 5. 사용자 관리 (ADM-USER-LIST)
> **핵심 정보:** 플랫폼 전체 사용자 현황 조회, 다중 처리 및 차단 상태 제어

### 1) 상단 필터 그룹
* **의미:** 전사 직원 타겟 유저를 찾기 위한 3중 필터 매트릭스 도구. (텍스트 검색 / 계정 상태 Select / 부서 필터)
* **동작:** 검색/조건 변경 시 하단 데이터는 Debounce 또는 `[조회]` 버튼 기반 실시간 갱신됨.
* **표시:** 헤더 아래 1행 가로 배치 폼 세트.

### 2) 상단 액션
* **의미:** 현황 보고서 생성을 돕는 원시 데이터 내보내기. (`📥 CSV 내보내기` 버튼)
* **동작:** 현재 필터링된 테이블 JSON을 CSV 스트림으로 파싱하여 다운로드.
* **표시:** 검색 필터열 우측 가장자리 버튼 UI.

### 3) 데이터 테이블 및 일괄 처리 (Bulk Action)
* **의미:** 직원 명단 마스터 데이터 목록.
* **동작:**
  * **검색/정렬:** 헤더 칼럼 클릭 시 실시간 정렬.
  * **일괄 동작:** 좌측 체크박스로 동시 선택 후 `[🚫 선택 차단]` 또는 `[✅ 선택 승인]` 누르면 일괄 변환(Confirm 모달 개입).
  * 개별 인원 `[상세]` 클릭 시 사용자 상세로 이동.
* **표시:** 표 영역 요소별(아바타 이미지, Badge) 심미성 부여 테이블 뷰어.

### 4) 하단 제어
* **의미:** 대량의 레코드 끊어 읽기 제어. (총 건수 및 페이지네이션)
* **동작:** Offset 번호 클릭 시 페이지 이동.
* **표시:** 좌측 `총 N건`, 우측 `[1] [2] [3]...` 렌더링.

## 6. 사용자 상세 (ADM-USER-DETAIL)
> **핵심 정보:** 특정 사용자의 제반 정보, 사용 트래픽 및 연동된 Key 세부 정보 심층 파악

### 1) 상단 액션
* **의미:** 의심 상황 판단 후 즉결 심판을 위한 기능. (`[차단]` 버튼)
* **동작:** 상태 차단 모달(ADM-USER-BLOCK) 발동.
* **표시:** 이름 옆 거부감 있는 붉은색 버튼 배치.

### 2) 기본 정보 / 관련 정보 패널 (4분할)
* **기본정보:** 성명, 이메일, 직원상태, 로그인 기록 텍스트 블록.
* **토큰 요약:** 월 예산 소모량을 시각화하는 Progress Bar 와 총비용($).
* **AI Tool 패널:** 현재 사용 인가된 Tool 목록 및 활성화 상태 테이블 표출.
* **API Key 매핑:** 관리자가 부여한 키 Alias 목록 로우 뷰잉 (열람 전용, Read Only).

## 7. 계정 차단/해제 (ADM-USER-BLOCK)
> **핵심 정보:** 위험 유저에 대한 직접적인 Account Blocking 폼 모달

### 1) 경고 배너 및 요약
* **의미:** 오인 사격 방지를 위한 이중 체크 구역.
* **표시:** `계정 차단 시 즉각 서비스 중단됨` 노란 경고 알럿 아래 대상 이름 명시.

### 2) 입력 폼 
* **의미:** 무단 제재 방지 및 사후 감사(Audit) 보고, 차단 당사자 알림용 시스템. (차단 사유 기재)
* **동작:** 입력 값 부재 시 버튼 비활성화 방어.
* **표시:** `사유를 구체적으로 기입` Placeholder가 세팅된 넓은 Textarea.

### 3) 액션 버튼
* **동작:** `[완료]` 버튼. 브라우저 Confirm 후 DB 상태 플래그 토글 및 호출 화면 복귀(Badge 변경 렌더링).

---

## 8. AI Tool 신청 관리 / 등록 로직 (ADM-APPLY-LIST & BATCH)
> **핵심 정보:** 사용자들의 AI Service 도입/제공 인가 요청 워크플로우 통제 및 공급 관리 전면 화면

### 1) 상단 상태 탭 / 등록 방식 라우터
* **의미:** 기안 처리 상태별 그룹핑 라우팅 및 대량/단건 매핑 라우팅 편의성.
* **동작/표시:** `[전체] / [대기중] / [승인] / [반려]` 탭 및 `단건 / 대량 등록 모드 전환` 탭 스위칭 지원.

### 2) 입력 폼 필드 구성 (단건)
* **플랫폼 드롭다운:** OpenAI, Google 등 제공사 고정값 선택.
* **API Key 입력칸:** 암호표기(password) 표출. 붙여넣기만 지원하며, 실수 방어용 재입력 필드구현.
* **내부 식별 폼:** 관리 목적의 키 이름과 Description 스트링.

### 3) 대량(Bulk) 엑셀 업로드 
* **의미:** 수십 개/다수 부서 할당을 돕는 이원화 시스템.
* **동작/표시:** 제공되는 .xlsx 템플릿(Key값, Provider, Alias 포맷) 다운로드 지원. 드래그앤드롭 직후 "총 14건 중 12건 성공 예감, 2건 구문 형식 오류" 사전 밸리데이션(Preview) 검증 출력 이후 Save.

## 14 & 15. API Key 관리 및 통계 / 파기 (ADM-PKEY-DETAIL / DELETE)
> **핵심 정보:** 계정 비용 지출 파악 및 비상시 킬 스위치(Kill Switch)

### 1) 상단 제어 조작
* **Active/Inactive State 스위치:** 토글 스위치 변경 시 즉시 [성공] 토스트와 함께 프록시 인입 하드 렌더링 차단 (소프트 삭제 체인).
* **`[삭제]` 액션:** 완전 영구 폐기용 버튼. (ADM-PKEY-DELETE 경보 진입)

### 2) 통계 대시보드 및 삭제 경고
* **통계 모니터링 뷰:** 그래프 등에 누수 및 비정상 어미징 여부 파악 뷰어 노출 (건수/토큰/오류율).
* **영향도 경고 (삭제 방어 체계):** 🚨 아이콘 옆, _"마이그레이션부터 하세요. 삭제하면 N명의 프롬프트가 응답 장애 처리됩니다"_ 위협 폰트 출력 및 타이핑형 서약("삭제함") Confirm 이중 안전장치 동작.

## 22. 요청/보안/감사 로그 통합 뷰어 (ADM-LOG-REQUEST / SECURITY / AUDIT)
> **핵심 정보:** 내부 보안 관제(SIEM 연동 이전) 및 사후 감사(Audit), 추적을 위한 3종 데이터 레코드 뷰어

### 1) [요청/응답] 로그 뷰어 (프롬프트 통신 원장 뷰어)
* **마스터 데이터 테이블:** 시간, 소속, 상태 표 등 필수 컬럼의 슬림한 1열 래핑.
* **아코디언(확장) 뷰어:** Data Row 앞단의 `[+]` 접기를 누르면 하단에 트래픽 패킷 뷰어가 노출됨.
  * _메타데이터 영역:_ UUID, 응답 딜레이, Target URL 텍스트 표기.
  * _Raw Payload 영역:_ 좌(사용자 프롬프트) - 우(LLM 응답) 2분할(병렬형) 텍스트 스크롤 컴포넌트로 대화 내역 전체 열람 지원.
  * _DLP 영역:_ 해당 패킷에 발생한 차단/마스킹 정보 박스 하단 첨부.

### 2) [보안 및 탐지] 관제탑 레이더
* **이벤트 요약:** 정상 통신은 버리고 'DLP / 보안 정책 / 비정상 접근 차단' 건만 필터링 된 뷰어.
* **위협 경고:** 최상단 빨강/노란 아이콘 블럭(Stat Card) 및 위반 행위에 대한 강렬한 라벨(Label) 마스킹 표시 렌더링.

### 3) [사내 관리자 감사 (Audit)] 시스템 변경 조작 이력
* **의미:** IT팀 어드민 본인들이 누굴 죽이고 누굴 엑세스 했는지 조작 이력을 남겨 내부 신뢰성을 지키는 장부.
* **표시:** 조작 실시자명, 당한 피의자명, 수행 액션(차단/할당), 접속 IP 등이 고딕계열 굵은 폰트 테이블 구조로 떨어져, 위조 방지형 원장(Ledger) 다운 시인성 강조 렌더링.
