## 🧠 secretary 메모리 업데이트 제안

**1. 새로운 사실: 외부 서비스 연동 장애 유형 확대**
- 기존의 권한 오류(Error 403) 외에도, 구글 캘린더의 `Read timeout`(응답 지연) 및 Gmail의 `JSON 파싱 오류`(데이터 구조 오류)가 발생하여 외부 데이터의 실시간 조회가 불안정한 상태임.

**2. 업무 노하우: 외부 연동 장애 시 '로컬 데이터 무결성 점검' 프로토콜**
- 외부 API(Google 등) 호출 실패 시, 외부 의존도를 낮추고 내부 시스템(Vault)의 자정 작용에 집중함:
    - **Inbox 관리**: `00_Inbox` 내에 처리되지 않고 방치된 파일이 있는지 전수 조사.
    - **Task 관리**: 완료(`Done`) 처리되지 않은 미완료 항목들을 찾아 업데이트.
    - **구조 정렬**: `Work`나 `Wealth` 폴더에 속해야 할 문서들이 루트 디렉토리에 방치되어 있지 않은지 폴더 계층 구조를 즉시 점검 및 정리.
