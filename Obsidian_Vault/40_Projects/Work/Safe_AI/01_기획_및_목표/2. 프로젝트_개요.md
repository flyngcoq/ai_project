# 🚀 프로젝트 개요: 개발자용 AI 프록시 서비스 (Design Thinking Approach)

## 1. Empathize (공감하기)
우리의 고객은 누구이며, 어떤 구체적인 페인 포인트(Pain Point)를 겪고 있는가?

### 👨‍💻 유저 페르소나 1: 사내 개발자 (Engineer)
* **상황:** 사내 시스템에 최신 AI 개발자툴(Cursor, Claude Code 등)과 AI 모델(GPT-4, Claude 등)를 연동해 빠르게 신기능을 출시하고 싶음.
* **Pain Point:** 
  * "방화벽 정책을 뚫고, 내부 보안 심사받고, 망분리 규제를 우회하는 아키텍처를 짜느라 실제 기능 개발보다 사전 작업이 훨씬 오래 걸립니다."
  * "모델마다 API 스펙이 다르고, 네트워크 에러나 타임아웃 처리를 프로젝트마다 일일이 구현하기 번거롭습니다."
  * "사내에서 AI 활용 시 사내 보안가이드나 정책이 부족하여 내가 보안에 맞게 잘 사용하고 있는지 모르겠고, 사용 시 문제가 발생할 것이 우려됩니다."
  * "AI 활용 능력이 나의 개인 업무역량이라고 생각하는데, 사내에서 AI 기능을 차단하여 성장에 정체되고 있다는 생각이 듭니다."

### 🛡️ 유저 페르소나 2: 보안/인프라 관리자 및 PM
* **상황:** 전사적으로 불어오는 AI 도입 열풍 속에서 보안 사고(데이터 유출)와 과도한 요금 폭탄을 막아야 함.
* **Pain Point:**
  * "누가 어떤 프롬프트에 사내 기밀이나 고객 개인정보(PII)를 넣어서 퍼블릭 LLM에 보내는지 파악할 길이 없습니다. (Shadow AI 통제 불가)"
  * "회사 공용 API Key가 팀 내 메신저에서 돌아다니고 있는데, 이 키가 외부에 탈취되면 모든 보안 책임은 제가 져야 합니다."
  * "AI 기능은 빠르게 변화/발전하여 보안 관련 확인할 요소는 많아지고 있는데, 보안정책 수립을 따라가기가 어렵습니다. 이에 따라, 안정적이고 지속적으로 검증이 가능한 보안 상품 발굴 또한 어렵습니다."
  * "부서별로 API를 얼마나 썼는지 투명한 비용 정산(Billing)이 불가능해 IT 예산 통제가 불가능합니다."

## 2. Define (문제 정의)
**"어떻게 하면 (How might we)..."**

개발팀이 겪는 **[망분리 등 복잡한 보안/인프라 절차, 파편화된 API 및 AI 개발 툴 연동의 어려움, 안전한 사용 가이드 부재로 인한 성장 정체]**를 완전히 해소하여 개발 생산성을 높이는 동시에,
보안/관리자가 최우선으로 요구하는 **[전사 AI 사용 가시성 확보, 기밀/개인정보 유출 원천 차단, 예측 가능한 예산 통제, 기술 발전에 맞춘 지속적이고 검증된 보안 정책의 수술 및 제공]**을 모두 충족하는 중앙 집중형 플랫폼을 만들 수 있을까?

## 3. Ideate (핵심 솔루션 및 기능 구체화)
위의 문제(Define)를 해결하기 위한 우리의 핵심 솔루션(MVP)

1. **통합 AI API 게이트웨이 및 개발 툴 연동 지원**
   * 복수의 LLM(OpenAI, Anthropic 등)을 단일 규격의 인터페이스 한 곳으로 모아 개발 파편화 방지.
   * Cursor, Claude Code 등 최신 AI 개발툴과 즉시 연동 가능한 표준화된 엔드포인트 제공.
   * 공통 네트워크 폴백(Fallback), 재시도(Retry), 타임아웃 처리를 프록시가 대행하여 개발 편의성 극대화.
2. **API Key 중앙화 및 안전한 자격 증명 (Secure RBAC)**
   * 개별 개발자에게는 사내 계정(SSO) 인증 토큰만 발급하고, 실제 외부 LLM API Key는 프록시 서버 내부에 안전하게 은닉. (키 탈취 원천 차단)
3. **권한 기반 토큰 할당 및 알림 대시보드 (Quota & Billing)**
   * 부서/프로젝트 생성 시 월별 최대 사용 한도(Budget Limit)를 하드 코딩하고, 80% 이상 소진 시 자동 이메일/슬랙 알림 및 차단.
4. **실시간 트래픽 감사(Audit) 및 데이터 마스킹 (Security/DLP)**
   * **(기본)** 모든 프롬프트/응답 데이터와 요청자 정보를 감사 로그(Audit Log)로 분리 보관하여 사후 추적(컴플라이언스) 지원.
   * **(고도화)** 정규식 매칭 및 필터링을 통한 사내 기밀, 주민번호 등 중요 PII 인라인 마스킹 기능.
5. **사내 AI 보안 가이드라인 제공 및 검증 체계 구축 (Governance)**
   * 개발자가 안심하고 사용할 수 있는 명확한 사내 AI 사용 정책/보안 가이드라인(Best Practice) 제공.
   * AI 기능 및 검증된 보안 아키텍처 등의 변화 동향을 지속 트래킹하여 신규 보안 위협 및 규제에 가장 빠르게 대응.
6. **망분리 해소 및 사전 승인된 보안 아키텍처 (Network & Compliance)**
   * **단일 방화벽 예외 처리 (Single Egress):** 프록시 서버에 대해서만 1회 방화벽을 열어두고, 개발자들은 사내 통신으로 프록시만 호출하면 되어 수십 개의 앱이 제각각 방화벽 심사를 받는 일이 없어집니다.
   * **망분리 우회 브릿지:** 사내 개발망과 외부 인터넷망을 이어주는 안전한 중계기로 동작하여, 규제를 위반하지 않고 망분리 환경에서도 외부 LLM 연동을 달성합니다.
   * **보안 패스포트 (Pre-approved Security):** 이미 로깅/마스킹 등 회사의 필수 보안 정책이 적용된 단위(프록시)를 경유하므로, 이를 사용하는 각 개발 프로젝트의 내부망 반출 및 연동 보안 심사가 대폭 통과/면제됩니다.

## 4. Value (최종 제공 가치)
* **개발 조직 타겟:** 안전하게 사전 세팅된 AI 통합 인프라와 권장 가이드라인 위에서 "버튼만 누르면" 최신 AI 개발툴과 API를 활용해 즉시 신기능 개발에 돌입. (Time-to-Market 극단적 단축 및 개인 역량 강화 지원)
* **관리/보안 조직 타겟:** 블랙박스와 같았던 사내 전역의 AI 활용 내역을 투명한 중앙 대시보드로 확인하고, 지속적으로 업데이트되는 보안 템플릿/정책 기반으로 빈틈없는 통제력(Governance) 확보.
