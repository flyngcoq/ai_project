## 2. 주요 구성요소 및 데이터 흐름

### 2.1 퍼블릭 구간 (접속 및 로드 밸런싱)
*   **User ➡️ Cloudflare:** 엔드유저의 모든 트래픽은 글로벌 라우팅 및 보안(DDoS 방어, WAF 등)을 거치기 위해 Cloudflare를 가장 먼저 통과합니다.
*   **Public Subnet 내 ALB (Application Load Balancer):** 트래픽이 AWS 환경으로 인입되는 유일한 퍼블릭 게이트웨이입니다. 트래픽의 목적지(사용자용 서버, 관리자용 서버 등)에 맞추어 Private Subnet 안의 자원들로 안전하게 라우팅합니다.

### 2.2 프라이빗 앱 영역 (핵심 컴퓨팅)
모든 비즈니스 로직과 컨테이너/인스턴스는 파블릭 IP를 갖지 않는 Private Subnet에 위치하여 외부의 임의 접근을 차단합니다.
*   **Amazon ECS (`usafeai-vibe`):** 컨테이너 기반으로 독립 실행되는 핵심 서비스 또는 부가적인 유틸리티 서비스 환경입니다.
*   **Auto Scaling group (`usafeai-admin`):** 관리자 기능 및 관제/모니터링을 담당하는 EC2 인스턴스 클러스터로, 트래픽에 맞춰 자동으로 스케일 아웃/인 됩니다.
*   **Auto Scaling group (`usafeai-user`):** 실제 사용자(고객)들의 API 요청 및 프록시 처리를 담당하는 핵심 EC2 인스턴스 클러스터입니다.
*   **EventBridge:** 특정 시간 스케줄, 또는 시스템 내의 상태 변화 발생 시 `usafeai-admin` 그룹 및 `usafeai-user` 그룹으로 트리거를 보내 자동화된 이벤트를 처리합니다.

### 2.3 프라이빗 데이터 및 보안 연동 (코어 스택)
실행된 인스턴스(ECS, EC2 등)들이 안전하게 공통 데이터를 저장하고 기밀 연동을 수행하는 컴포넌트입니다.
*   **PostgresDB / DynamoDB:** 정형 및 비정형 데이터를 저장하는 프라이빗 데이터베이스 층입니다.
*   **Secret Manager:** 암복호화 키, 외부에 제공하는 API 토큰 등 하드코딩해서는 안 되는 민감한 자제 증명 값들을 안전하게 저장 및 제공합니다.
*   **Amazon S3 ➡️ SIEM (Splunk):** 시스템 상에서 발생하는 중요한 감사(Audit) 로그나 보안 트래픽 덤프 파일 등을 S3 버킷에 적재합니다. S3에 저장된 로그들은 서드파티 통합 로그 모니터링 시스템인 **Splunk(SIEM)**로 안전하게 아웃바운드 포워딩되어 24시간 보안 관제 운영에 활용됩니다.

### 2.4 외부 AI 모델 아웃바운드 연동
*   **NAT Gateway ➡️ Internet Gateway:** Private Subnet 내의 앱 서버들은 인터넷과 직접 통신할 수 없으므로, 아웃바운드 트래픽에 한해 NAT 게이트웨이를 거쳐 인터넷 게이트웨이(IGW)로 빠져나갑니다.
*   **AI Tools (Codex, Claude):** Safe AI의 핵심 기능인 거대 언어 모델(LLM) 처리를 위해 OpenAI, Anthropic의 외부 API Endpoints로 패킷을 전송하고 응답을 받아옵니다.
