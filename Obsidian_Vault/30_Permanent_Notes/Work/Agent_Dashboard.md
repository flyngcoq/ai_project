# 🧭 멀티 에이전트 대시보드 (Agent Dashboard)
📅 최종 업데이트: 2026-05-06

본 문서는 현재 시스템에서 구동 가능한 AI 페르소나와 그 활용법을 정리합니다. 텔레그램에서 `@이름`을 입력하여 해당 에이전트를 소환할 수 있습니다.

---

## 🤖 활성 에이전트 목록 (Agent Directory)

| 소환 태그 | 에이전트명 | 주요 역할 | 활용 예시 |
| :--- | :--- | :--- | :--- |
| **@ceo** | **CEO (기본)** | 총괄 지휘, 전략 수립, 답변 최적화 | "현재 우리 비즈니스의 가장 큰 리스크는?" |
| **@developer**| **Developer** | 코드 작성, 버그 수정, 시스템 아키텍처 | "telegram_bot.py에 새로운 핸들러 추가해줘" |
| **@researcher**| **Researcher** | 정보 검색, 데이터 수집, 트렌드 분석 | "최근 AI 에이전트 트렌드에 대해 조사해줘" |
| **@secretary** | **Secretary** | 일정 관리, 간단한 메모, 인박스 정리 | "내일 오전 10시에 미팅 일정 잡아줘" |
| **@writer** | **Writer** | 블로그 포스팅, 이메일 작성, 창의적 글쓰기 | "이번 주 자산 보고서를 블로그 톤으로 써줘" |
| **@editor** | **Editor** | 문서 교정, 톤앤매너 수정, 요약 | "이 보고서의 가독성을 높여줘" |
| **@designer** | **Designer** | UI/UX 제안, 그래픽 디자인 가이드 | "대시보드에 어울리는 색상 팔레트 추천해줘" |
| **@business** | **Business** | 비즈니스 모델링, 수익성 분석 | "새로운 상가 투자 수익률 시뮬레이션해줘" |

---

## 🛠️ 에이전트 커스터마이징 (Customization)
각 에이전트의 지능과 기억은 `_agents/` 폴더 내의 파일들을 수정하여 고도화할 수 있습니다.

*   **`prompt.md`**: 에이전트의 말투, 전문 지식, 구체적인 지시 사항을 정의합니다.
*   **`memory.md`**: 해당 에이전트가 반드시 기억해야 할 고유한 히스토리를 기록합니다.
*   **`config.md`**: (시크릿) 에이전트별 전용 API 키 등을 설정합니다.

---

## 💡 똑똑하게 굴리는 팁
1.  **페르소나 전환**: "이 코드를 **@developer**가 검토하고, **@editor**가 요약해줘"와 같이 한 문장에서 여러 역할을 지정할 수 있습니다.
2.  **기본값**: 별도의 태그가 없으면 가장 똑똑한 **@ceo**가 응답합니다.
3.  **지식 결합**: 모든 에이전트는 사용자님의 **옵시디언 지식 베이스**를 공통적으로 참고하므로, 개인 자산 데이터를 기반으로 한 특화된 답변이 가능합니다.
